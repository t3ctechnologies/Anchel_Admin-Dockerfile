Domain models

After having to create a connection to an LDAP directory, it is necessary to create a domain model.

This template will allow you to define the correspondence between the first name, last name and email fields. {{ 'COMMON.NAME' | translate }}, and the attributes of your directory. {{ 'COMMON.NAME' | translate }} requires 4 different queries to work properly:

  • LQL query (Ldap Query Language) authentication: query to find a user in the directory during the authentication phase from the identifier that he has entered (email)
  • LQL query (Ldap Query Language) user : query to find a user in the directory             when searching for consultation, editing, sharing, ...
  • LQL query (Ldap Query Language) autocomplete on all attributes: allows the autocompletion of a             user from the entered motif
  • LQL query (Ldap Query Language) autocompletion on first and last name: allows autocompletion of a user             from a couple of reasons: first-name or first-name
